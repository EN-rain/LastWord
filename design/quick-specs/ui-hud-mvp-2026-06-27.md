# Quick Design Spec: UI & HUD MVP Polish + First-Time Setup

**Type**: New Small System (with Tuning/Tweak sub-components)
**System**: UI & HUD (§11)
**GDD Reference**: `GAME_DESIGN.md` §11.2 Persistent HUD Elements, §11.3 Accessibility Features, §11.4 First-Time Setup Screen
**Date**: 2026-06-27
**Estimated Implementation**: 6–8 hours

## Overview

This spec bundles the highest-value, lowest-dependency UI/HUD gaps from §11 so the vertical slice can reach a playable state without waiting for art, Steam, or post-launch systems. It covers:

1. HUD phase tracker (top-left three-dot progress).
2. Token indicator polish (proximity pulse rate tiers + Tier-1 light-blue rule).
3. Voice activity indicator colour fix (Tier 1 = light blue).
4. Listener proximity pulse accessibility feature (screen-edge pulse within 12 m).
5. First-Time Setup screen (region, calibration, accessibility, keybind review, privacy acknowledgement).

Items explicitly **out of scope** for this spec: battery indicator (torch item not implemented), teammate status icons (death/spectator loop incomplete), Listener direction arc (depends on full Listener position sync polish), subtitle system (no SFX assets/events), keybind remapping UI already exists in `SettingsMenu`, Text Broadcaster gameplay mode (depends on `TextBroadcaster.cs`).

## Core Rules

### Phase Tracker

- Anchored top-left of the HUD, three dots representing Phase 1, Phase 2, Phase 3.
- Completed phases: filled white dot.
- Current phase: faint glow / larger scale.
- Future phases: empty outline.
- On `GameManager.PhaseChanged`, update immediately.
- If `GamePhase.Victory`, all three dots fill gold.
- If `GamePhase.Failed`, current dot turns red and pulses once.

### Token Indicator Pulse Rates

- The top-centre skull/TextureRect pulses when the Token is held.
- Pulse rate depends on distance from local player to **Listener**:
  - `> 12 m`: slow pulse (2.0 s period).
  - `8–12 m`: medium pulse (1.0 s period).
  - `4–8 m`: fast pulse (0.5 s period).
  - `< 4 m`: rapid strobe (0.2 s period).
- Distance is measured from local `PlayerController` to the single `ListenerAI` in scene.
- If no Listener exists, use slow pulse as safe default.
- If Token holder is in Silence Room, hide indicator (exception: Frenzy targets — not implemented yet, see deferred).
- Indicator appears only after first speech / Token transfer, not at run start.

### Voice Activity Indicator Colours

- Tier 0 Silent: grey.
- Tier 1 Whisper: **light blue** (`(0.4, 0.75, 0.95, 1)`).
- Tier 2 Normal: orange.
- Tier 3 Loud: red.
- Apply to both `TierLabel` font and `VolumeMeter` tint.

### Listener Proximity Pulse

- Screen-edge colour pulse when Listener is within 12 m of local player.
- Toggle stored in `user://settings.cfg` under `settings/proximity_pulse`.
- Off by default; enabled from First-Time Setup or Settings > Accessibility.
- Warm colour (amber/red) that intensifies as Listener gets closer.
- Implemented as a full-screen `ColorRect` vignette modulated by alpha.
- Update at 2 Hz to keep cost low.
- Suppressed when player is dead/spectating.

### First-Time Setup Screen

- Auto-shown before Main Menu if `settings/setup_complete` is missing/false.
- Steps shown in sequence (Next/Back navigation):
  1. **Region selection** — dropdown with NA East, NA West, EU West, EU Central, SEA, OCE. Stored in `settings/region`.
  2. **Mic calibration** — reuses `SettingsMenu` in simplified mode; mandatory; cannot skip.
  3. **Accessibility toggles** — Proximity Pulse, Subtitles (stored only), Text Broadcaster (stored only). All off by default.
  4. **Keybind review** — read-only display of default controls; states remapping is available in Settings.
  5. **Privacy acknowledgement** — full Early Access notice; explicit “I understand” button required to finish.
- After completion, set `settings/setup_complete = true` and `settings/gdpr_accepted = true` with timestamp.
- Values persist to `user://settings.cfg`.

## Tuning Knobs

| Knob | Default | Range | Category | Rationale |
|---|---|---|---|---|
| Token slow pulse period | 2.0 s | 0.1–5.0 s | feel | Distant threat, low urgency |
| Token medium pulse period | 1.0 s | 0.1–5.0 s | feel | Moderate urgency |
| Token fast pulse period | 0.5 s | 0.1–5.0 s | feel | High urgency |
| Token rapid strobe period | 0.2 s | 0.05–2.0 s | feel | Immediate danger |
| Proximity pulse max distance | 12.0 m | 6–24 m | gate | Matches accessibility threshold in GDD |
| Proximity pulse update rate | 2 Hz | 1–10 Hz | performance | Keeps UI cheap |

All values live as `[Export]` fields on the relevant scripts so designers can tune in-editor.
